# 用户指南

本文档面向日常使用者，介绍子非鱼安全工具箱的主要界面、常用操作和运行逻辑。

## 1. 界面概览

主界面由三部分组成：

- 左侧：一级分类
- 中间：二级分类
- 右侧：工具卡片区

顶部工具栏包含常用入口：

- 新建工具
- 搜索
- 主题
- 配置
- 刷新
- 收藏
- 笔记
- 体检
- 关于

程序启动后默认会优先进入收藏页；如果当前没有收藏工具，则按当前数据状态展示工具列表。

## 2. 新建与编辑工具

点击“新建工具”后，可以配置以下信息：

- 工具类型：本地工具 / 网页工具
- 名称
- 路径或 URL
- 描述
- 一级分类 / 二级分类
- 是否收藏
- 图标
- 运行命令 / 参数
- 自定义类型标签
- 是否在终端中运行

说明：

- 网页工具填写 `http://` 或 `https://` 地址
- 本地工具填写可执行文件、脚本或目录路径
- 对于网页工具，程序会尝试根据 URL 下载站点 favicon 作为图标
- 图标保存在运行时目录下的 `.runtime/resources/icons/`

## 3. 工具启动逻辑

程序会根据工具类型与配置自动决定启动方式。

### 3.1 网页工具

- 使用系统默认浏览器打开

### 3.2 本地目录

- 打开该目录

### 3.3 本地文件

- GUI 工具通常直接打开
- 命令行工具可结合“运行命令 / 参数”和“在终端中运行”进行启动

### 3.4 终端启动

当工具勾选“在终端中运行”时：

- “打开工具”会按终端工具方式处理
- `运行命令 / 参数` 字段用于填写要执行的命令
- 可使用 `{path}` 占位符引用当前工具路径

如果未手动填写工作目录，程序通常会自动推导为工具所在目录。

## 4. 分类与收藏

### 4.1 分类浏览

- 支持一级分类和二级分类浏览
- 可新建分类与子分类
- 可删除分类或子分类

### 4.2 收藏页

- 可以将工具加入收藏
- 点击顶部“收藏”进入独立收藏布局
- 收藏页适合放置高频使用工具

## 5. 搜索

顶部搜索框用于全局搜索。

当前搜索会匹配：

- 工具名称
- 工具描述
- 工具 Markdown 笔记内容

搜索结果会统一显示在右侧工具区域，并根据匹配强度排序。

## 6. Markdown 笔记与附件

每个工具都可以绑定独立 Markdown 笔记。

支持：

- 新建和编辑工具笔记
- Markdown 预览
- 笔记列表查看
- 笔记全文搜索
- 附件归档

笔记与附件默认保存在：

- `.runtime/resources/notes/`
- `.runtime/resources/notes/_attachments/`

笔记文件会以稳定标识 `tool_<id>.md` 保存；旧格式笔记会在需要时迁移。

## 7. 主题

当前内置多套主题，包括：

- 亮绿色主题
- 蓝白色主题
- 紫霓主题
- 红橙主题

主题选择会写入运行时 `settings.ini`，下次启动时自动恢复。

## 8. 配置菜单功能

顶部“配置”菜单包含以下功能：

### 8.1 导入配置

导入本工具箱导出的配置文件。

### 8.2 导出配置

导出当前工具配置，适合做配置迁移或备份。

### 8.3 导入天狐2.0

支持导入天狐 2.0 工具箱导出的 JSON 配置。

导入后会：

- 尝试根据当前分类体系重映射分类
- 将无法识别的工具放入 `待分类（天狐导入）`
- 为导入工具标记来源，便于后续清理

### 8.4 删除天狐2.0导入

一键删除所有天狐导入来源的工具，不影响手工添加或本工具箱原生导入的数据。

### 8.5 同步官方工具库

支持从配置中指定的远程 JSON 地址同步官方工具库。

此功能依赖 `settings.example.ini` / `settings.ini` 中的同步配置，建议在发布版本中根据实际验证结果使用。

### 8.6 检查更新 / 一键更新

程序内置更新检查与一键更新入口。

如果发布版本启用了对应发布源与更新包，可通过该入口检查新版本并执行更新。若你的发布方式仅为源码运行，请以实际发布说明为准。

## 9. 数据体检

点击顶部“体检”可打开数据体检入口。

它适合用于这些场景：

- 导入大量工具后
- 准备发布仓库前
- 怀疑数据存在失效路径、分类错配、重复项或资源异常时

仓库级发布检查还可以使用：

```powershell
python scripts/repo_sanity_check.py
```

## 10. Windows 桌面快捷方式

如果你希望从桌面直接启动程序，可运行：

```powershell
python scripts/create_desktop_shortcut.py
```

脚本会在当前用户桌面创建一个 `.lnk` 快捷方式，优先尝试使用可用的 `pythonw.exe` 启动程序。

## 11. 常见问题

### 工具点开没有反应

优先检查：

- 路径是否有效
- 是否误把目录当成文件配置
- 是否需要勾选“在终端中运行”
- 工作目录是否需要手动指定

### 搜索不到工具

优先检查：

- 关键词是否过短
- 工具名称或描述是否真实包含目标词
- 目标内容是否只存在于笔记中

### 发布到 GitHub 后别人无法直接运行你的工具配置

这是正常现象。仓库中的 `data/tools.json` 应该保留模板数据或占位数据，个人本地路径不适合直接发布。推荐让用户在自己的运行环境中重新配置本地工具路径。
